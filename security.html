Security & Code Review

Every AI-generated line deserves a second look. Here are the free tools that do it automatically.

🚫
Never Do These
  • Paste real API keys, passwords, or tokens into any AI chat window
  • Commit a .env file (add it to .gitignore before creating it)
  • Run AI-generated code without reading every line first
  • Accept all AI suggestions blindly — understand before you commit
  • Store credentials directly in config files that go into Git

Sample .coderabbit.yaml for Class Repos

language: "en-US"
reviews:
  profile: "assertive"
  high_level_summary: true
  auto_review:
    enabled: true
    base_branches:
      - "main"
path_filters:
  ignore:
    - "**/*.min.js"
    - "node_modules/**"
instructions:
  - "This is a student project at IndigoSpace Padang, Indonesia."
  - "Focus on teaching moments — explain why, not just what."
  - "Flag any hardcoded credentials or API keys as critical."
  - "Check for missing input validation on all user inputs."

  semgrep:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: semgrep/semgrep-action@v1
        with:
          config: p/security-audit

Oops — I Committed a Secret

Act fast. Bots scan GitHub in real time.

1

Rotate the Secret NOW

Go to the service (AWS, GitHub, Stripe) and revoke/regenerate the key. Do this first, before anything else. Assume the old key is already compromised.

↓
2

Remove from History with BFG

brew install bfg   # Mac
echo "YOUR_SECRET" > secrets.txt
bfg --replace-text secrets.txt my-repo.git
cd my-repo && git reflog expire --expire=now --all
git gc --prune=now && git push --force --all
↓
3

Notify Collaborators

Anyone who cloned the repo must re-clone. History was rewritten — their local copy still has the secret.

↓
4

Prevent Recurrence

pip install detect-secrets pre-commit
detect-secrets scan > .secrets.baseline
pre-commit install

Pre-commit hooks block secrets before they're ever committed.
